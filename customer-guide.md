# 全部文档 - 小二群发器

## 软件介绍

小二群发器是一款 Telegram 多账号运营工具，适合用于账号管理、群组消息发送、批量私信、群成员采集、群组采集、筛群、账号预热和私信监听。

当前版本：V2.6.6

本文档主要介绍软件下载、激活、基础设置和各功能的使用方法。

## 一、下载与安装

### 1. 下载软件

请通过客服或管理员提供的下载地址获取最新版软件。

下载完成后，请确认文件没有被浏览器或安全软件拦截。如果下载失败，可以更换网络后重试，或联系客服获取备用下载方式。

### 2. 启动软件

下载完成后，双击运行软件。

首次启动可能会稍慢，属于正常情况。如果系统弹出安全提示，选择允许或继续运行即可。

### 3. 输入卡密激活

打开软件后，输入客服提供的卡密。

验证成功后即可进入主界面。

如果提示验证失败，请检查：

- 卡密是否输入正确。
- 卡密是否已经到期。
- 当前设备是否已更换。
- 网络是否正常。
- 是否正在使用过期版本。

如果确认输入无误仍无法进入，请联系客服处理。

## 二、账号管理

账号管理是所有功能的基础，用于导入、登录、检测和维护 Telegram 账号。

### 主要功能

- 导入 Telegram Session。
- 登录新账号。
- 批量检测账号状态。
- 导出选中账号。
- 删除异常账号。
- 批量修改账号资料。
- 踢出其他登录设备。
- 一键修复异常账号。

### 使用步骤

1. 打开账号管理页面。
2. 点击导入 Session 或登录账号。
3. 登录账号时按提示输入手机号、验证码和两步验证密码。
4. 导入完成后点击检测状态。
5. 只选择状态正常的账号执行后续任务。

### 状态说明

| 状态 | 说明 |
| --- | --- |
| 在线 | 账号正常，可以使用。 |
| 未登录 | Session 失效，需要重新登录。 |
| 双向 | 账号通信受限，建议减少任务或暂停使用。 |
| 冻结 | 账号异常，需要重新处理。 |
| 封禁 | 账号已不可用，建议移除。 |

### 使用建议

- 任务开始前先检测账号状态。
- 不要把所有任务集中在一个账号上。
- 异常账号应及时暂停使用。
- 定期导出正常账号备份。

## 三、群发功能

群发功能用于向指定群组或频道发送消息，支持多账号、多目标、多轮次执行。

### 支持模式

| 模式 | 说明 |
| --- | --- |
| 直发 | 直接发送文本或格式化内容。 |
| 转发 | 从指定消息来源转发内容。 |
| 引用 | 引用指定消息并发送回复内容。 |
| Post | 以频道消息形式发送。 |
| 贴纸 | 发送贴纸内容。 |
| 名片 | 发送用户或群组名片。 |

### 使用步骤

1. 选择要参与发送的账号。
2. 添加目标群组或频道链接。
3. 选择发送模式。
4. 填写发送内容或来源消息。
5. 设置发送间隔、群组间隔、轮次和定时发送。
6. 点击开始群发。
7. 在任务日志中查看成功和失败数量。

### 使用建议

- 发送间隔不要设置过短。
- 建议先小批量测试，再批量执行。
- 同一内容不要在短时间内重复发送过多。
- 多账号执行时要合理分配任务量。

## 四、批量私信

批量私信用于向用户列表发送私信内容。

### 主要功能

- 导入用户名或用户 ID。
- 从采集结果导入目标用户。
- 使用文案模板。
- 设置发送间隔。
- 支持定时发送。
- 记录发送成功和失败结果。

### 使用步骤

1. 选择发送账号。
2. 导入目标用户列表。
3. 输入私信内容或选择模板。
4. 设置发送间隔和失败重试。
5. 点击开始发送。
6. 根据日志查看发送结果。

### 使用建议

- 私信内容尽量自然，不要过度重复。
- 控制单账号每日发送数量。
- 遇到账号受限时应停止该账号任务。

## 五、批量进群验证

批量进群验证用于多个账号批量加入群组，并处理部分入群验证流程。

### 支持模式

| 模式 | 说明 |
| --- | --- |
| 均分模式 | 将群组平均分配给多个账号。 |
| 全量模式 | 每个账号都尝试加入全部群组。 |

### 使用步骤

1. 选择进群模式。
2. 添加群组链接。
3. 选择执行账号。
4. 根据需要配置自动验证接口。
5. 点击开始进群。
6. 查看加入成功、已在群内、失败和需人工处理的记录。

### 使用建议

- 加群速度不要过快。
- 新账号建议先预热后再执行大量加群。
- 验证失败的群组可单独人工处理。

## 六、监听私信

监听私信用于实时查看账号收到的私信，可配合关键词过滤使用。

### 主要功能

- 多账号监听私信。
- 关键词过滤。
- 查看发送者信息。
- 查看消息内容和接收时间。
- 可按需设置自动回复。

### 使用步骤

1. 选择要监听的账号。
2. 设置关键词，留空则显示全部私信。
3. 点击开始监听。
4. 在监听列表查看实时消息。
5. 不需要监听时点击停止。

### 使用建议

- 只监听需要处理的账号，减少资源占用。
- 关键词应设置得清晰，避免误命中过多。

## 七、强拉成员

强拉成员用于把目标用户邀请到指定群组。

### 使用步骤

1. 选择目标群组。
2. 导入目标用户列表。
3. 选择执行账号。
4. 设置邀请间隔。
5. 点击开始执行。
6. 查看成功、失败和受限记录。

### 注意事项

- 请确保账号具备对应邀请权限。
- 邀请频率不宜过高。
- 失败用户可以导出后单独处理。

## 八、群成员采集

群成员采集用于从指定群组中采集成员用户名、用户 ID、昵称等信息。

### 使用步骤

1. 添加目标群组。
2. 选择已加入目标群组的账号。
3. 设置采集数量和过滤条件。
4. 点击开始采集。
5. 采集完成后导出 TXT 或 CSV 文件。

### 常用过滤条件

- 用户名过滤。
- 昵称关键词过滤。
- 活跃状态过滤。
- 重复用户过滤。

### 使用建议

- 大群采集耗时较长，请耐心等待。
- 多个账号可分担不同群组。
- 导出结果前建议先去重。

## 九、群采集

群采集用于通过关键词搜索或频道内容提取 Telegram 群组链接。

### 采集方式

| 方式 | 说明 |
| --- | --- |
| 机器人搜索 | 使用搜索机器人按关键词采集群组。 |
| 频道消息提取 | 从频道历史消息中提取群组链接。 |

### 使用步骤

1. 选择采集方式。
2. 输入关键词或频道地址。
3. 设置翻页数量和最大采集数量。
4. 点击开始采集。
5. 导出群组链接。

### 使用建议

- 多准备几个关键词，采集结果会更完整。
- 使用前建议先筛选无效链接。

## 十、筛群功能

筛群用于分析群组活跃度，帮助筛选质量更高的目标群。

### 使用步骤

1. 导入待筛选群组。
2. 选择用于检测的账号。
3. 设置采样数量和活跃标准。
4. 点击开始筛群。
5. 根据活跃度结果保留高质量群组。

### 结果说明

| 等级 | 说明 |
| --- | --- |
| 高 | 活跃用户比例较高，适合优先使用。 |
| 中 | 活跃度一般，可视情况使用。 |
| 低 | 活跃度较低，建议谨慎使用。 |

## 十一、账号预热

账号预热用于让账号逐步进入正常使用状态，适合新号或长时间未使用的账号。

### 预热阶段

| 阶段 | 建议时间 | 内容 |
| --- | --- | --- |
| 阶段 1 | 1-3 天 | 保持在线，浏览消息。 |
| 阶段 2 | 4-7 天 | 增加查看群消息、点击内容等行为。 |
| 阶段 3 | 8-14 天 | 少量互动、少量加群。 |
| 阶段 4 | 15-21 天 | 提高使用频率。 |
| 阶段 5 | 22-30 天 | 接近正常使用强度。 |

### 使用建议

- 新账号建议从阶段 1 开始。
- 预热期间不要执行高频任务。
- 设置合理工作时间，避免全天运行。

## 十二、配置管理

配置管理用于设置 Telegram API、代理、任务频率、工作时间和自动验证接口。

### 常见配置

| 配置项 | 说明 |
| --- | --- |
| API ID | Telegram API ID。 |
| API Hash | Telegram API Hash。 |
| 代理设置 | SOCKS5、HTTP 或其他网络代理。 |
| 发送间隔 | 群发、私信、加群等操作间隔。 |
| 每日数量 | 单账号每日任务上限。 |
| 工作时间 | 软件允许执行任务的时间段。 |
| 自动验证接口 | 用于处理部分入群验证。 |

### 使用建议

- API ID 和 API Hash 请妥善保存。
- 代理要稳定，否则会影响账号连接。
- 任务频率应根据账号质量调整。
- 修改配置后，部分设置需要重启软件生效。

## 十三、广告文字格式

发送内容支持常用格式标签，方便制作更清晰的消息内容。

### 常用标签

| 标签 | 说明 |
| --- | --- |
| [加粗]内容[/加粗] | 加粗文字。 |
| [斜体]内容[/斜体] | 斜体文字。 |
| [下划线]内容[/下划线] | 下划线文字。 |
| [删除线]内容[/删除线] | 删除线文字。 |
| [链接=https://example.com]文字[/链接] | 添加链接。 |
| [换行] | 换行。 |
| [代码]内容[/代码] | 行内代码样式。 |
| [引用]内容[/引用] | 引用样式。 |

### 示例

```text
[加粗]限时活动[/加粗]
[换行]
[链接=https://example.com]点击查看详情[/链接]
```

### 注意事项

- 标签要成对使用。
- 链接需要以 http 或 https 开头。
- 发送前建议先小范围测试显示效果。

## 十四、软件更新教程

软件启动后会自动检查是否有新版本。

如果检测到新版本，请按提示下载并安装。

### 强制更新

如果提示必须更新，请先下载最新版后再继续使用。旧版本可能无法继续进入软件。

### 下载失败怎么办

如果下载失败，请检查：

- 网络是否正常。
- 下载地址是否可以打开。
- 本地安全软件是否拦截下载。
- 当前卡密是否仍在有效期内。

仍无法下载时，请联系客服获取备用下载地址。

## 十五、常见问题

### 1. 软件打不开怎么办？

请确认文件是否下载完整，是否被安全软件拦截。如果仍然打不开，请联系客服重新获取最新版。

### 2. 卡密验证失败怎么办？

请检查卡密是否输入正确、是否过期、是否已经在其他设备使用。如果确认无误仍失败，请联系客服处理。

### 3. 为什么运行中提示授权失效？

可能是卡密已到期、设备状态变化、网络长时间无法连接服务器，或当前版本需要更新。请联系客服检查。

### 4. 为什么提示需要更新？

说明当前版本不是最新版。请按提示下载新版安装包后继续使用。

### 5. 更新下载失败怎么办？

请更换网络重试，或联系客服获取备用下载链接。

### 6. 账号为什么显示双向、冻结或封禁？

这是 Telegram 账号自身状态异常。建议暂停该账号任务，必要时重新登录或更换账号。

### 7. 群发或私信失败很多怎么办？

建议先检查账号状态，再降低发送频率，减少单账号任务量，并确认目标群组或用户是否有效。

## 十六、使用建议

- 始终使用最新版软件。
- 不要共享卡密。
- 不要频繁更换设备。
- 任务开始前先检测账号状态。
- 群发、私信、加群都要设置合理间隔。
- 遇到异常提示及时联系客服。
